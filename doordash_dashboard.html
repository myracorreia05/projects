Experiment design
A/B test simulator
Statistical power guide
Min sample size for 80% power at α=0.05
Experiment significance over time
Simulated p-value convergence
P-value convergence for experiment 1 over time.
DoorDash uses switchback tests
Standard user-level A/B fails in marketplaces — one user's order affects another dasher's availability. DoorDash's engineering blog documents time-based switchback splits as their primary experiment design.
300 orders/arm for 80% power
To detect a 5% AOV lift at α=0.05 with 80% power, each arm needs ≥280 orders. Our dataset satisfies this for every zone-level test.
Surge pricing: p=0.023 ✓
Experiment 1 is the strongest result — +$4.80 AOV with only 1.2% conversion loss. The p-value of 0.023 crosses the α=0.05 threshold on day 14.

About this project
What makes this different
Most portfolios show bar charts and stop there. This project mirrors the exact analytics stack documented in DoorDash's engineering blog — cohort retention heatmaps, switchback A/B experiment design, Z-score anomaly detection, and demand-based price elasticity modeling.

Every analysis is grounded in published data science methodology. The ML section includes production-ready scikit-learn code with proper cross-validation, SMOTE for class imbalance, and reproducible random seeds. The dashboard is a single self-contained HTML file — zero dependencies beyond Chart.js, deployable on GitHub Pages in 30 seconds.
